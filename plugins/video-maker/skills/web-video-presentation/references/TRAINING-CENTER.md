# 培训中心 / Training Center 风格（结构驱动）

这不是"换个语气的平台变体"，而是一种**结构驱动的内容体裁**：把一个产品 /
主题做成**分层赋能课程**——目标是让学习者**获得能力**（能讲清、能上手、能做
方案），不是娱乐。选了这个风格，口播稿、章节结构、视觉都按"认证 / 培训课"
的惯例走。

> **🔒 核心隔离原则（最重要）**：本框架是从一个"解决方案顾问认证课"的例子里
> 提炼出来的**产品无关骨架**。任何示例产品（API 网关、数据库、SaaS、硬件…）
> 只是用来说明骨架长什么样 —— **绝不把任何示例产品写进用户的 script /
> outline / 章节**。用户带来什么产品，就用他的产品填这套 L0–L3，示例产品
> 一个字都不许漏进产物。你要学的是"骨架和风格"，不是某个具体产品。

---

## 和 B 站 / 抖音风格的根本区别

| 维度 | 娱乐向（B站 / 抖音…） | 培训中心 |
|---|---|---|
| 受众 | 泛观众，图一乐 / 涨点知识 | 被赋能者（partner / 销售 / 顾问 / 新人），要拿到能力 |
| 目标 | 看完爽、记住一个点 | 学完**能做某事**（能讲清、能上手、能做方案） |
| 语气 | 自来熟、段子、强钩子 | 专业、清晰、有指导性，像资深同事带教；**不用网络梗 / 夸张钩子**，但要有"你将学会"的路径驱动 |
| 结构 | 钩子 → 展开 → 金句 | **目标先行 → 能力边界 → 逐层递进 → 阶段小结** |
| 密度 | 一个 idea 慢慢讲 | 信息密度偏高、结构化，但仍**一知识点一 step** 逐步揭示 |
| 落点 | 情绪 / 认同 | **能力获得 + 可操作**（方法 / 清单 / playbook） |

## L0–L3 能力分层框架（视频内容；产品无关）

把任意产品 / 主题套进这四层。每一层是一级**能力台阶**，不是固定章节清单——
具体模块由产品决定。

| 层 | 定位（学完这层能…） | 该讲什么（产品无关） |
|---|---|---|
| **L0 · 导学 / Onboarding & Scope** | 知道这门课 / 这条视频带你到哪、不覆盖什么 | 课程价值与目标、**能力边界**（明确不讲什么）、面向谁、学习路径与时长预算 |
| **L1 · 基础认知 / Foundation** | 和团队"说同一种话"、建立心智模型 | 领域 / 行业背景、核心术语（中英对照）、产品的**核心架构与对象模型**（由哪些东西组成、彼此什么关系） |
| **L2 · 核心能力 / Core Capabilities** | 知道产品"能干什么"、每项能力怎么用 | 把产品能力切成若干**核心模块**，每模块讲：是什么 / 解决什么问题 / 关键机制 / 怎么用 / **能力边界**。一个模块 = 一项核心能力 |
| **L3 · 方案方法 / Solution Methods** | 从"懂产品"到"会做方案" | 典型场景与行业打法（playbook）、售前 / 落地方法论（如 发现需求 → 画蓝图 → 做验证）、把前面的能力组合成端到端方案 |

> **L4（认证考试）与 EXT（配套资产 / 工具包）不做成视频** —— 它们是课程基建
> （考核、模板、问题库、参考图库），不是口播视频的内容。需要时另行准备。

逐层是**递进**关系：L0 定边界 → L1 给地基 → L2 铺能力 → L3 教组合落地。
做单条视频时不必覆盖全部四层（见下「两种粒度」）。

## 两种视频粒度（开工前先问用户这条是哪种）

选了培训中心风格后，**先确认这条视频是"课程总览 / 导学"还是"单模块深讲"**——
两者章节结构不同：

### A. 课程总览 / 导学（一条视频 = 整门课的体系图）

把整套 L0–L3（可顺带提及 L4 认证）讲成一张"能力地图 + 学习路径"。建议章节：

1. **开场**：这门课解决谁的什么问题、学完你能做什么（价值 + 目标）
2. **能力边界**：覆盖什么 / 不覆盖什么（防止预期错位）
3. **能力地图**：L0→L3 各层讲什么（每层一屏，逐层揭示），标模块与时长感
4. **学习路径**：建议顺序、难度递进、总时长
5. **产出与认证**：学完拿到什么（能力 / 证书 / 工具包），收尾召唤行动

### B. 单模块深讲（一条视频 = 某一层里的一个模块）

聚焦一个模块（通常在 L1 / L2 / L3），按"教学闭环"走。建议章节：

1. **模块目标 + 能力边界**：这一节学完你能做什么、不讲什么
2. **背景 / 前置**：为什么需要它、它在产品里的位置（接 L1 的心智模型）
3. **核心知识点**：逐层展开（一知识点一 step），关键机制配可视化演示
4. **典型场景 / 示例**：把知识点放进真实场景演一遍（示例随客户产品换）
5. **小结 + "你现在能做什么"**：回扣目标，给可操作的 checklist / 方法
6. **（可选）自测**：一两道题，引到下一模块

## 培训中心的口播 / 表达约定

- **目标先行**：每段开头点明"学完这段你能做什么"；结尾给"现在你能…"。
- **能力边界要说出来**：明确不覆盖什么，避免学习者预期错位。
- **中英双语标注**：模块名 / 关键术语首次出现给中英对照（企业 / 海外 partner
  培训惯例）。正文语言跟 [`SKILL.md`](../SKILL.md) Phase 1.1.5 确认的成片语言。
- **时长预算**：总览里给每模块 / 每层的时长感（培训课惯例，帮学习者规划）。
- **逐步揭示**：密度高但不堆砌——清单 / 步骤 1 项 = 1 step，关键机制用
  CSS / SVG / 动画演示（遵守 [`CHAPTER-CRAFT.md`](CHAPTER-CRAFT.md) 的视觉演示
  与反 AI 味红线）。
- **专业但不干**：像资深同事带教的节奏，不用段子也不照本宣科；术语解释清楚、
  例子具体。
- **视觉**：建议配套 `training-center` 主题（净白 + royal 蓝 + 左侧层级条 +
  高密度），气质最贴。

## 反模式（培训中心特有）

- ❌ **把示例产品（如 API 网关）写进产物** —— 框架产品无关，客户的产品才是内容。
- ❌ **写成营销话术 / 产品宣传片** —— 这是赋能课，目标是"学会"，不是"种草"。
- ❌ **堆术语不解释 / 跳过能力边界** —— 学习者会预期错位、跟不上。
- ❌ **把 7~8 小时整门课塞进一条视频** —— 一条 = 一个模块或一张总览图，不是整门课。
- ❌ **丢掉"目标 / 小结"** —— 培训视频的闭环靠"目标先行 + 学完能做什么"撑起来。
